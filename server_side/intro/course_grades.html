course_grades

Description

This table provides the grading event of when the user reached his or her highest grade in one course (of any branch of the course, see course_branch_grades for specific branch grades). Also provides the passing/not-passing state for each user and course.

Columns

NameDescription
course_id22 character unique string identifying an individual course
ualberta_user_idEncrypted Coursera user id for ualberta data.
course_grade_tsThe timestamp for when a learner's course grade has changed. This occurs everytime a grading event takes place.
course_passing_state_idA learner can be in one of four "states" with respect to passing a course: 0 means they have started the course but not passed, 1 means they have completed the course without identity verification, 2 means they've completed the course with identity verification, 3 means the course is not passable.
course_grade_overall_passed_itemsThe number of items in the course that the learner has passed.
course_grade_overallThe grade a learner has in the course, according to the instructor-set grading policy, based on passed items.
course_grade_verified_passed_itemsThe number of items in the course that the learner has passed and verified their identity.
course_grade_verifiedThe grade a learner has in the course, according to the instructor-set grading policy, based on items that have been passed and for which the learner's identity has been verified.

SQL create statement

CREATE TABLE course_grades (
    course_id varchar 
    ,ualberta_user_id VARCHAR(50) NOT NULL
    ,course_grade_ts timestamp 
    ,course_passing_state_id int4 
    ,course_grade_overall_passed_items int4 
    ,course_grade_overall float4 
    ,course_grade_verified_passed_items int4 
    ,course_grade_verified float4 
    ,PRIMARY KEY (course_id, ualberta_user_id, course_grade_ts)
    ,FOREIGN KEY (course_passing_state_id) REFERENCES course_passing_states(course_passing_state_id)
);
